Robert Brandom’s (2008) Incompatibility Semantics

Definitions and Axioms

We are given a language \(L\) (set of sentences). \(L\) is proper if, for each \(p \in L\) and each \(q\) a subformula of \(p\), we also have \(q \in L\). All languages under consideration are proper.

Axiom: Persistence

Description

If a set of sentences is incoherent (Inc), adding more sentences to it cannot make it coherent.

Formal Statement

\[\label{eq:persistence} \forall X, Y \subseteq L, (X \subseteq Y \wedge X \in \text{Inc}) \rightarrow Y \in \text{Inc}\]

Rationale

Incoherence, once established, cannot be resolved by merely adding more sentences. Contradictions require removal or correction of conflicting elements.

Axiom: Partition

Description

Two sets of sentences are incompatible (\(I\)) if and only if their union is incoherent (\(\text{Inc}\)).

Formal Statement

\[\label{eq:partition} X \cup Y \in \text{Inc} \iff X \in I(Y)\]

Rationale

Incompatibility between sets is equivalent to their union being incoherent. Combining incompatible statements leads to contradiction.

Entailment

Description

One set of sentences entails (\(\models_I\)) another if everything incompatible with the conclusion is incompatible with the premise.

Formal Statement

\[\label{eq:entailment} X \models_I Y \iff \bigcup_{p \in Y} I(\{p\}) \subseteq I(X)\]

Rationale

Entailment means the truth (or commitment to) \(X\) guarantees the truth (or commitment to) \(Y\). If anything contradicts \(Y\), it must also contradict \(X\).

Connective Definitions

Axioms governing \(N\) (Negation), \(K\) (Conjunction), and \(L\) (Necessity).

Axiom: Negation Introduction (NI)

Description

Introducing negation (\(N\)) based on incoherence. \(Np\) means "not \(p\)".

Formal Statement

\[\label{eq:ni} X \cup \{Np\} \in \text{Inc} \iff X \models \{p\}\]

Rationale

Adding \(Np\) yields incoherence iff \(X\) entails \(p\). Reflects contradiction arising from \(p\) and \(\neg p\).

Axiom: Conjunction Introduction (CI)

Description

Introducing conjunction (\(K\)) based on incoherence. \(Kpq\) means "both \(p\) and \(q\)".

Formal Statement

\[\label{eq:ci} X \cup \{Kpq\} \in \text{Inc} \iff X \cup \{p, q\} \in \text{Inc}\]

Rationale

Conjunction \(Kpq\) leads to incoherence iff combining \(X\) with both \(p\) and \(q\) does. Reflects that conjunction is true iff both conjuncts are.

Axiom: L Introduction (LI)

Description

Introducing necessity (\(L\)). \(Lp\) means "necessarily \(p\)".

Formal Statement

\[\label{eq:li} X \cup \{Lp\} \in \text{Inc} \iff X \in \text{Inc} \text{ or } \exists Y [X \cup Y \not\in \text{Inc} \text{ and } Y \not\models \{p\}]\]

Rationale

\(Lp\) leads to incoherence with \(X\) if \(X\) is already incoherent, or if there’s a consistent extension \(Y\) of \(X\) that doesn’t entail \(p\). Captures necessity’s link to consistency across contexts.

Basic Lemmas

Lemma: Weakening

Description

Adding assumptions and conclusions preserves entailment (\(\models\)).

Formal Statement

\[\text{If } X \models Y, \text{ then } X, W \models Y, V\]

Rationale

Reflects monotonicity: more information doesn’t invalidate previous inferences.

Lemma: Cut

Description

Transitivity of entailment.

Formal Statement

\[\text{If } X \models q, Y \text{ and } q, W \models V, \text{ then } X, W \models Y, V\]

Rationale

Allows breaking complex inferences into steps via intermediate statements.

Some Modal Properties

Lemma 3.1

Description

Condition relating non-entailment and non-tautology.

Formal Statement

\[\exists Y [Y \not\models \emptyset \text{ and } Y \not\models p] \iff \not\models p\]

Rationale

If there’s a consistent context \(Y\) not entailing \(p\), then \(p\) is not universally entailed (not a tautology).

Lemma 3.2

Description

Condition relating necessity and inconsistency.

Formal Statement

\[Lp \models \emptyset \iff \models \emptyset \text{ or } \not\models p\]

Rationale

\(Lp\) entails inconsistency iff the base system is inconsistent or \(p\) is not universally entailed. Necessity’s failure can signal inconsistency or contingency.

Reduction Schemata for Non-Modal Connectives

Left Negation (LN)

Description

Negation on the left.

Formal Statement

\[\label{eq:ln} X, Np \models Y \iff X \models Y, p\]

Rationale

Moves negation from premise-side to conclusion-side.

Right Negation (RN)

Description

Negation on the right.

Formal Statement

\[\label{eq:rn} X \models Y, Np \iff X, p \models Y\]

Rationale

Moves negation from conclusion-side to premise-side.

Left Conjunction (LK)

Description

Conjunction on the left.

Formal Statement

\[\label{eq:lk} X, Kpq \models Y \iff X, p, q \models Y\]

Rationale

Breaks down conjunction in premises.

Right Conjunction (RK)

Description

Conjunction on the right.

Formal Statement

\[\label{eq:rk} X \models Y, Kpq \iff X \models Y, p \text{ and } X \models Y, q\]

Rationale

Distributes entailment over conjunction in conclusions.

Reduction Schemata for Modal Connectives

Left Necessity (LL)

Description

Necessity on the left.

Formal Statement

\[\label{eq:ll} X, Lp \models Y \iff X \models Y \text{ or } \not\models p\]

Rationale

Necessity in premise adds a strong condition related to universal entailment of \(p\).

Right Necessity (RL)

Description

Necessity on the right.

Formal Statement

\[\label{eq:rl} X \models Y, Lp \iff X \models Y \text{ or } \models p\]

Rationale

Entailing necessary \(p\) requires either entailing \(Y\) or \(p\) being universally entailed.

Validity of Classical Logic

The reduction schemata show that the incompatibility semantics validates classical logic for \(N\) and \(K\), and the modal system S5 for \(L\).

Mathematics

Von Neumann ordinals provide a foundational method for representing natural numbers within set theory. They are adopted here as a normative standard.

Definition 1 (Von Neumann Ordinal). Each natural number is represented as the set of all smaller natural numbers: \[0 = \varnothing, \quad 1 = \{0\} = \{\varnothing\}, \quad 2 = \{0, 1\} = \{\varnothing, \{\varnothing\}\}, \quad \ldots\]

Construction requires the empty set, guaranteed by axioms (e.g., in ZF or KP). The empty set \(\varnothing = \{ x \mid x \neq x \}\) aligns with the normative understanding that objects cannot possess mutually incompatible properties (). \(\varnothing\) serves as a foundational building block.

Von Neumann Ordinals
Number Successor Representation Representation as Sets
0 {} \(\varnothing\)
1 \(\{0\}\) \(\{\varnothing\}\)
2 \(\{0, 1\}\) \(\{\varnothing, \{\varnothing\}\}\)
3 \(\{0, 1, 2\}\) \(\{\varnothing, \{\varnothing\}, \{\varnothing, \{\varnothing\}\}\}\)

Note: Von Neumann ordinals used as the normative standard for numbers.

Mathematical Judgment

Foundational judgment involves **Particular Instantiations (PINs)** of properties (). E.g., "PINA of \(P\) is distinct from PINB of \(P\)."

Example:

  1. Pokey and Buddy are distinct dogs.

  2. Implicitly: Pokey is a dog.

Recognizing "Pokey is a dog" relies on intersubjective agreement within a recognitive community.

The "I-think"

Every judgment involves the subject’s awareness. Representing this formally:

  1. Pokey is a dog.

  2. \(\vdash\) Pokey is a dog.

  3. {Pokey is a dog}.

  4. {Pokey \(\varnothing\) is a dog}.

  5. I said \(\varnothing\) ‘Pokey \(\varnothing\) is a dog’.

  6. I said \(\varnothing\) ‘I said \(\varnothing\) ‘Pokey \(\varnothing\) is a dog’.’

  7. Pokey and Buddy are {\(\varnothing\), {\(\varnothing\)}} distinct dogs.

  8. Pokey and Buddy are 2 distinct dogs.

Here, \(\varnothing\) marks the subject’s role. Numerical terms emerge from relational properties of judgments, aligning with von Neumann ordinals: \(2 \leftrightarrow \{\varnothing, \{\varnothing\}\}\).

Anaphoric Number and Successor Function

Numbers act anaphorically, referencing prior judgments. The sequence (2)-(8) reflects the successor function inherent in von Neumann ordinals.

Reduction Schemata

  • Left Negation (LN): \(X, Np \models Y \iff X \models Y, p\)

  • Right Negation (RN): \(X \models Y, Np \iff X, p \models Y\)

  • Left Conjunction (LK): \(X, Kpq \models Y \iff X, p, q \models Y\)

  • If \(\{p\} \in I(X)\) and \(\{Np\} \in I(X)\), then \(X \in \text{Inc}\).

  • \(NNp \approx p\). (Double Negation)

  • \(p \models q \iff Nq \models Np\). (Contraposition)

  • Material Consistency: \(I(X) \subseteq \text{Inc} \Rightarrow \exists Y [Y \in I(X) \text{ and } I(Y) \subseteq \text{Inc}]\).

  • Formal Consistency: \((X \notin I(X) \text{ and } X \models p) \Rightarrow X \not\models Np\).

  • \(Kpq \models p\) and \(Kpq \models q\).

  • \((X \models p \text{ and } X \models q) \iff X \models Kpq\).

Soundness, Completeness, and Compactness

Soundness

Description

If derivable, then true in all models.

Formal Statement

\(\text{If } \vdash p, \text{ then } \models p\). (Using \(\vdash\) for derivability)

Rationale

System reliability: does not produce falsehoods.

Completeness

Description

If true in all models, then derivable.

Formal Statement

\(\text{If } \models p, \text{ then } \vdash p\).

Rationale

System capability: derives all truths.

Compactness

Description

If every finite subset of \(X\) is satisfiable, then \(X\) is satisfiable.

Formal Statement

Standard definition.

Rationale

Crucial for infinite sets/structures.

Highlander Automaton

Purpose

Models gradual elimination until one candidate remains. Start state = Accept state symbolizes unity of action.

Bounded Region Automaton

Purpose

Purpose
